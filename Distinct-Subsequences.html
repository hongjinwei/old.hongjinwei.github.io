Distinct Subsequences

@https://leetcode.com/problems/distinct-subsequences/

Given a string S and a string T, count the number of distinct subsequences of T in S.

A subsequence of a string is a new string which is formed from the original string by deleting some (can be none) of the characters without disturbing the relative positions of the remaining characters. (ie, "ACE" is a subsequence of "ABCDE" while "AEC" is not).

Here is an example: S = "rabbbit", T = "rabbit"

Return 3.

HashMap<String, String> err = new HashMap<String, String>();

public boolean isErr(String s, String t) {
    if( s.length() == 0 || t.length() == 0 || s == null || t == null) return false;
    return err.containsKey(s) && err.get(s).equals(t);

public boolean process(String s, String t) {
    if(t.length() == 0 || t == null) {
        count++;
        return true;
    }
    if(s.length() == 0 || s == null || s.length() < t.length()) return false ;

    boolean flag = false;
    for(int i = 0; i < s.length(); i++){
        if( s.charAt(i) == t.charAt(0) && !isErr( s.substring(i+1), t.substring(1)) ){
            if( !process( s.substring(i+1), t.substring(1) ) ){
                for(int j = i+2; j<=s.length(); j++){
                    err.put( s.substring(i+1, j), t.substring(1) );
                    err.put( s.substring(i+1, j), t);
                }
            }else{
                flag = true;
            }
        }
    }
    return flag;
}

然后，妥妥的超时了。在String很长的时候，递归太深，导致结果很慢。而且必须利用hashmap来做剪纸，不然效率还要低下。

这道题，如果用DP来做的话，效率要高好多，那么该如何寻找状态转移方程呢？

首先，举例来看，当找到了一个匹配的字母的时候：

dda da 此时，第一个匹配了  
^   ^

dda da 继续，第二个字母与第一个字母匹配
 ^  ^

dda da 当匹配第二个的时候， 由观察我们可以知道，由于前面有2个匹配，因此此时第二个可匹配的方式有2种。
  ^  ^

我们可以看出，当匹配 t 中第 i 个字母的时候,有多少种可能和前一个字母有多少种可能有关（ 实际上多出了前一个字母的可能总数 ）。

因此，我们扫描每一个s 中的字母，每个字母分别与 t 中每一个字母比对，如果相等，则这个字母的可能数就加上前一个字母的可能数。如果是第一个（index = 0），那么就+1.

另外还需要注意的是，必须从 t 的尾部向前扫描，因为如果从前往后扫描，如果 t 中有字母重复，那么就会影响后面的结果。

从前往后扫描（Wrong way！）： 

ddd dd    ans[0] = 1;
^   ^

ddd dd    ans[1] += ans[0]; (ans[1] = 1);
^    ^ 

ddd dd    ans[0] = 2;
 ^  ^

ddd dd    ans[1] = 3;
 ^   ^

ddd dd    ans[0] = 3;
  ^ ^

ddd dd    ans[1] = 6;
  ^  ^

从后往前（Right way!）:

ddd dd    ans[1] = 0;
^    ^

ddd dd    ans[0] = 1;
^   ^

ddd dd    ans[1] = 1;
 ^   ^

ddd dd    ans[0] = 2;
 ^  ^

ddd dd    ans[1] = 3;
  ^  ^

ddd dd    ans[0] = 3;
  ^ ^

从上面的扫描过程我们可以看到，如果从前往后扫描，由于会改变之前的状态，会导致重复。

最后看AC代码：

public class Solution {
    public int numDistinct(String s, String t) {
        int[] ans = new int[t.length()];
        for (int i = 0; i < s.length(); i++) {
            char tmp = s.charAt(i);
            for(int j = t.length()-1; j >= 0; j--){
                if( tmp == t.charAt(j)){
                    if(j == 0){
                        ans[0]++;
                    }else{
                        ans[j] += ans[j-1];
                    }
                }
            }
        }
        return ans[t.length()-1];
    }  
}
